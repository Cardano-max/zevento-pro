# Plan 07.2-04: Admin Dashboard Complete — Enterprise Admin Panel

## Goal
Make every admin page production-ready with real API data, actionable controls, and a professional enterprise look.

## Context
- apps/admin: Next.js, Tailwind, has pages: dashboard, vendors, users, categories, commission-rates, leads, markets, payments
- Admin API endpoints at /admin/*
- Auth: JWT with ADMIN role
- Existing: dashboard (analytics), vendors (list/status), users, etc.

## Tasks

### 1. Admin Dashboard (apps/admin/src/app/page.tsx) — Enterprise Metrics
Full analytics dashboard:
- Top metrics row: Total Users, Total Vendors, Active Vendors, Total Bookings, Total Revenue
- Conversion funnel visualization (lead → quote → booking → payment)
- Revenue by stream table (subscription, commission, etc.)
- Leads per city bar chart (CSS-based)
- Recent activity feed: last 10 platform events
- Quick actions: Approve Vendors, View Reports, Moderate Feed

### 2. Vendors Page (apps/admin/src/app/vendors/page.tsx) — Enhanced
Already functional. Enhance:
- Vendor detail expansion: click row → expand panel with full info
- Approve/Reject/Suspend buttons with confirmation dialog
- KYC status badge
- Filter by role (PLANNER, SUPPLIER, VENDOR)
- Search by business name
- View vendor profile link

### 3. Users Page (apps/admin/src/app/users/page.tsx) — Complete
- List all users (GET /admin/users)
- Show: name, phone, email, roles, status, joined date
- Filter by role (CUSTOMER, VENDOR, ADMIN)
- Search by phone/name
- Deactivate/Reactivate user toggle
- View user details

### 4. Feed Moderation Page (apps/admin/src/app/feed/page.tsx) — New
Admin feed moderation:
- List all feed posts (GET /feed?status=all with admin token)
- Show post body, author, status badge
- Hide/Unhide post button → PATCH /admin/feed/:id/hide
- Delete post → DELETE /admin/feed/:id  
- View comments for each post
- Filter by status (ACTIVE | HIDDEN | REPORTED)

### 5. Reports Page (apps/admin/src/app/reports/page.tsx) — New
Moderation reports:
- List all reports (GET /admin/reports)
- Show: reporter name, target type, target ID, reason, status, date
- Review button → mark REVIEWED
- Action button → suspend target account + mark ACTIONED
- Dismiss button → mark DISMISSED
- Filter by status (PENDING | REVIEWED | ACTIONED)

### 6. Bookings Page (apps/admin/src/app/bookings/page.tsx) — New
Full booking oversight:
- List all bookings (GET /admin/bookings or /booking/all)
- Show: booking ID, customer, vendor, event date, status, amount
- Status filter (CONFIRMED, IN_PROGRESS, COMPLETED, CANCELLED, DISPUTED)
- Click → booking detail modal with full timeline

### 7. Payments/Transactions Page (apps/admin/src/app/payments/page.tsx) — Enhanced
Already exists. Enhance:
- Show transaction list with type, amount, status
- Filter by type (SUBSCRIPTION, COMMISSION, DEPOSIT, FULL_PAYMENT)
- Revenue summary at top
- Export CSV button (placeholder)

### 8. Admin Layout Improvements
