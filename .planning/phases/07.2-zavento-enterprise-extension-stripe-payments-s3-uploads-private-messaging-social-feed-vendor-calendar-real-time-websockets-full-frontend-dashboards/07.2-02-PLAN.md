# Plan 07.2-02: Customer Web Enterprise Features — Feed + Favorites + Inbox Pages

## Goal
Add the Social Feed page, Favorites page, and Inbox/Messaging page to the customer web app (apps/web). Also enhance the vendor detail page with favorites toggle and social share.

## Context
- apps/web uses Next.js 15, Tailwind v4, Zustand, TanStack-style api() calls
- API proxy at /api/proxy routes all calls to backend
- Feed API: GET /feed (public), POST /feed (JWT), etc.
- Favorites API: GET/POST/DELETE /customer/favorites/:vendorId
- Existing pages: /, /login, /vendors, /vendors/[id], /plan, /dashboard

## Tasks

### 1. Social Feed Page (apps/web/src/app/feed/page.tsx)
Full feed page with:
- Header with gradient bg, "Wedding Community Feed" title
- Tab switcher: All | Requirements (CUSTOMER) | Vendor Offers
- City filter dropdown
- Post cards showing:
  - Author avatar (initials) + name + role badge (Customer/Vendor)
  - Post body (truncated to 3 lines)
  - Category tag, city, event date if present
  - Budget badge if present
  - Comments count
  - "View Comments" toggle
  - Report button (kebab menu)
- Create Post button (if logged in) → modal/sheet to create post
  - textarea, category select, city, event date, budget
- Pagination (load more)
- Loading skeletons

### 2. Create Post Modal Component
- Opens on "Share to Community" button
- Fields: Body (required), Category (REQUIREMENT/OFFER/SHOWCASE/GENERAL), City, Event Date, Budget
- POST /feed endpoint
- Shows success toast, refreshes feed

### 3. Favorites Page (apps/web/src/app/favorites/page.tsx)
- Auth-gated (redirect to /login)
- Grid of saved vendors (same VendorCard style as /vendors)
- Empty state with "Browse Vendors" CTA
- Remove from favorites button on each card
- GET /customer/favorites

### 4. Inbox Page (apps/web/src/app/inbox/page.tsx)
- Auth-gated
- Simple inbox showing all lead assignment conversations
- Left panel: conversation list (vendor name, last message preview, status badge)
- Uses GET /leads/inquiries to get the customer's inquiries
- Each inquiry card with vendor assignments shown
- "Contact Vendor" button linking to vendor profile
- Note: Real-time chat is in vendor app; customer inbox shows inquiry status

### 5. Enhance Vendor Detail Page — Favorites Toggle
Update apps/web/src/app/vendors/[id]/page.tsx:
- Add favorites heart button in hero section
- On click: POST /customer/favorites/:id or DELETE /customer/favorites/:id
- Check initial state: GET /customer/favorites/:id/check
- Show filled heart if favorited, outline if not
- Auth gate: redirect to login if not logged in

### 6. Update Navbar
Update apps/web/src/components/navbar.tsx:
- Add "Feed" link → /feed
- Add "Saved" link → /favorites (only if logged in)
- Keep existing links: Home, Vendors, AI Planner

### 7. Add Feed Link to Home Page
Update apps/web/src/app/page.tsx:
- Add "Community Feed" section between categories and featured vendors
- Show 3 most recent feed posts (GET /feed?limit=3)
- "See all posts" → /feed

## Styling Notes
- Use same gradient-bg class for headers
- Use same card styling as dashboard/vendors pages
- Vendor role badge: rose/red tones; Customer badge: blue/purple tones
- Feed should feel like a community board, not a social media timeline
