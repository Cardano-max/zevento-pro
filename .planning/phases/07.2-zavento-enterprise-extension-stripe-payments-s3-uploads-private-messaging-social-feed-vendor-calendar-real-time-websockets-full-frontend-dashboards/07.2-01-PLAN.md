# Plan 07.2-01: Backend Enterprise Schema Extensions + Feed + Favorites + Reports API

## Goal
Add Social Feed, Favorites, and Reports/Moderation to the Prisma schema and create backend API modules for all three, so the frontend can consume them.

## Context
- Existing schema: User, VendorProfile, Booking, Review, Lead, Inbox, BlockedDate, etc.
- Missing models: FeedPost, FeedComment, FavoriteVendor, Report
- Backend: NestJS at api/src/
- All new modules must be registered in AppModule

## Tasks

### 1. Schema Extensions (api/prisma/schema.prisma)
Add to schema.prisma (at the end, with Phase 07.2 comment):

```prisma
// ── Phase 07.2: Enterprise Extension ──

model FeedPost {
  id          String        @id @default(uuid()) @db.Uuid
  authorId    String        @map("author_id") @db.Uuid
  authorRole  String        @map("author_role") @db.VarChar(20) // CUSTOMER | VENDOR
  title       String?       @db.VarChar(200)
  body        String        @db.Text
  category    String        @default("GENERAL") @db.VarChar(30) // REQUIREMENT | OFFER | SHOWCASE | GENERAL
  mediaUrls   Json?         @map("media_urls") // string[]
  city        String?       @db.VarChar(100)
  eventDate   DateTime?     @map("event_date")
  budgetPaise Int?          @map("budget_paise")
  status      String        @default("ACTIVE") @db.VarChar(20) // ACTIVE | HIDDEN | REPORTED | DELETED
  likesCount  Int           @default(0) @map("likes_count")
  createdAt   DateTime      @default(now()) @map("created_at")
  updatedAt   DateTime      @updatedAt @map("updated_at")

  author      User          @relation("AuthorFeedPosts", fields: [authorId], references: [id], onDelete: Cascade)
  comments    FeedComment[]
  reports     Report[]      @relation("FeedPostReports")

  @@index([authorId])
  @@index([status, createdAt(sort: Desc)])
  @@index([category])
  @@map("feed_posts")
}

model FeedComment {
  id        String   @id @default(uuid()) @db.Uuid
  postId    String   @map("post_id") @db.Uuid
  authorId  String   @map("author_id") @db.Uuid
  body      String   @db.Text
  status    String   @default("ACTIVE") @db.VarChar(20)
  createdAt DateTime @default(now()) @map("created_at")

  post      FeedPost @relation(fields: [postId], references: [id], onDelete: Cascade)
  author    User     @relation("AuthorFeedComments", fields: [authorId], references: [id], onDelete: Cascade)

  @@index([postId])
  @@map("feed_comments")
}

model FavoriteVendor {
  id         String        @id @default(uuid()) @db.Uuid
  customerId String        @map("customer_id") @db.Uuid
  vendorId   String        @map("vendor_id") @db.Uuid
  createdAt  DateTime      @default(now()) @map("created_at")

  customer   User          @relation("CustomerFavorites", fields: [customerId], references: [id], onDelete: Cascade)
  vendor     VendorProfile @relation("VendorFavorites", fields: [vendorId], references: [id], onDelete: Cascade)

  @@unique([customerId, vendorId])
  @@index([customerId])
  @@map("favorite_vendors")
}

model Report {
  id           String    @id @default(uuid()) @db.Uuid
  reporterId   String    @map("reporter_id") @db.Uuid
  targetType   String    @map("target_type") @db.VarChar(30) // VENDOR | FEED_POST | MESSAGE | USER
  targetId     String    @map("target_id") @db.Uuid
  reason       String    @db.VarChar(100)
  description  String?   @db.Text
  status       String    @default("PENDING") @db.VarChar(20) // PENDING | REVIEWED | ACTIONED | DISMISSED
  adminNote    String?   @map("admin_note") @db.Text
  reviewedBy   String?   @map("reviewed_by") @db.Uuid
  reviewedAt   DateTime? @map("reviewed_at")
  createdAt    DateTime  @default(now()) @map("created_at")

  reporter     User      @relation("UserReports", fields: [reporterId], references: [id], onDelete: Cascade)
  feedPost     FeedPost? @relation("FeedPostReports", fields: [targetId], references: [id])

  @@index([status])
  @@index([targetType, targetId])
  @@map("reports")
}
```

Also add relations to existing models:
- User: add `feedPosts FeedPost[] @relation("AuthorFeedPosts")`, `feedComments FeedComment[] @relation("AuthorFeedComments")`, `favorites FavoriteVendor[] @relation("CustomerFavorites")`, `reports Report[] @relation("UserReports")`
- VendorProfile: add `favorites FavoriteVendor[] @relation("VendorFavorites")`

### 3. Create Feed Module (api/src/feed/)
Files:
- `feed.module.ts` — imports PrismaModule, exports FeedService
- `feed.service.ts` — CRUD for posts + comments
- `feed.controller.ts` — REST endpoints
- `dto/create-post.dto.ts`
- `dto/create-comment.dto.ts`

**Feed Service methods:**
- `getPosts(query: { category?, city?, page?, limit?, authorId? })` — paginated public feed
- `createPost(authorId, authorRole, dto)` — create feed post
- `deletePost(id, userId)` — soft delete (set status DELETED)
- `getComments(postId)` — comments for a post
- `createComment(postId, authorId, dto)` — add comment
- `deleteComment(id, userId)` — soft delete comment

**Feed Controller routes:**
- `GET /feed` — public feed (no auth required)
- `POST /feed` — create post (JWT required)
- `DELETE /feed/:id` — delete own post (JWT required)
- `GET /feed/:id/comments` — get comments (public)
- `POST /feed/:id/comments` — add comment (JWT required)
- `DELETE /feed/comments/:id` — delete own comment

### 4. Create Favorites Module (api/src/customer/favorites.controller.ts)
Add to existing customer module:
- `GET /customer/favorites` — list my favorites (JWT + CUSTOMER role)
- `POST /customer/favorites/:vendorId` — add favorite
- `DELETE /customer/favorites/:vendorId` — remove favorite
- `GET /customer/favorites/:vendorId/check` — check if favorited
