# Phase 07.2 Plan 01: Backend Enterprise Schema Extensions + Feed + Favorites + Reports API Summary

**One-liner:** NestJS social feed with paginated posts/comments, customer favorites with composite-unique idempotency, and moderation reports — all wired to Prisma schema with 4 new models and admin feed moderation endpoints.

## What Was Built

### Schema Extensions (api/prisma/schema.prisma)
Added 4 new models at end of schema with Phase 07.2 comment block:

- **FeedPost** — social/marketplace posts with category (REQUIREMENT/OFFER/SHOWCASE/GENERAL), status (ACTIVE/HIDDEN/REPORTED/DELETED), author relation, likes count, optional event date and budget
- **FeedComment** — post comments with soft-delete status field
- **FavoriteVendor** — customer-vendor favorites with `@@unique([customerId, vendorId])` composite constraint
- **Report** — moderation reports targeting VENDOR/FEED_POST/MESSAGE/USER entities, with admin review workflow

Relations added to existing models:
- **User**: feedPosts, feedComments, favorites, reports
- **VendorProfile**: favorites

Migration file created at: `api/prisma/migrations/20260315000000_enterprise_feed_favorites_reports/migration.sql`

### Feed Module (api/src/feed/)
- `GET /feed` — public paginated feed, filterable by category/city/authorId
- `POST /feed` — create post (JWT required, uses activeRole as authorRole)
- `DELETE /feed/:id` — soft-delete own post (JWT required)
- `GET /feed/:id/comments` — public comments list
- `POST /feed/:id/comments` — add comment (JWT required)
- `DELETE /feed/comments/:id` — soft-delete own comment (JWT required, route ordered before `/:id`)
- Admin via FeedService: `toggleHidePost`, `adminDeletePost`

### Favorites Endpoints (customer module extension)
- `GET /customer/favorites` — list my favorited vendors (JWT)
- `POST /customer/favorites/:vendorId` — add favorite, ConflictException if duplicate (JWT)
- `DELETE /customer/favorites/:vendorId` — remove favorite (JWT)
- `GET /customer/favorites/:vendorId/check` — check if favorited, returns `{ isFavorited: boolean }` (JWT)

### Report Module (api/src/report/)
- `POST /reports` — submit report (JWT required)
- `GET /admin/reports` — list reports paginated with status filter (ADMIN only)
- `PATCH /admin/reports/:id` — review/action report, sets reviewedBy + reviewedAt (ADMIN only)

### Admin Feed Moderation (api/src/admin/)
- `PATCH /admin/feed/:id/hide` — toggles ACTIVE/HIDDEN status
- `DELETE /admin/feed/:id` — hard-deletes post
- FeedModule imported into AdminModule so FeedService is injectable

### AppModule
FeedModule and ReportModule registered in imports array.
