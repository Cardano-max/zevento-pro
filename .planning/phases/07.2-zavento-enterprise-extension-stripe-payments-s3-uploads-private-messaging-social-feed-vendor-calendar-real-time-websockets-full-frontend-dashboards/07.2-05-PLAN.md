# Plan 07.2-05: Deploy All Three Apps + API to Production

## Goal
Deploy all platform apps to production: customer web to Vercel, vendor dashboard to Vercel, admin dashboard to Vercel, and push API changes to Render via GitHub.

## Tasks

### 1. Deploy API Changes (GitHub → Render auto-deploy)
```bash
cd /Users/mac/Desktop/zavento
git add api/
git commit -m "feat(07.2): enterprise extension - feed, favorites, reports API"
git push origin master
```

### 2. Deploy Customer Web App (apps/web → Vercel)
```bash
cd /Users/mac/Desktop/zavento/apps/web
npx vercel --prod --yes
```

### 3. Deploy Vendor Dashboard (apps/vendor → Vercel)
```bash
cd /Users/mac/Desktop/zavento/apps/vendor
npx vercel --prod --yes
```

### 4. Deploy Admin Dashboard (apps/admin → Vercel)
```bash
cd /Users/mac/Desktop/zavento/apps/admin
npx vercel --prod --yes
```

### 5. Run Build Checks Before Deploy
For each app, run:
```bash
cd apps/web && npm run build 2>&1
cd apps/vendor && npm run build 2>&1
cd apps/admin && npm run build 2>&1
```
Fix any TypeScript/build errors before deploying.

### 6. Verify Deployments
- Test all deployed URLs
- Check API connectivity (proxy working)
- Check auth flow (OTP login)
- Check feed page loads
- Check vendor dashboard loads

## Notes
- Each app deploys independently to separate Vercel projects
- API on Render auto-deploys from GitHub master push
- CORS already configured to allow *.vercel.app domains
- Check build logs if any deploy fails
