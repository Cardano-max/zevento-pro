    **File: api/src/inbox/inbox.gateway.ts**
    Implement InboxGateway with:
    - `@WebSocketGateway({ cors: { origin: '*', credentials: true } })` — NO port argument (shares HTTP port)
    - Implements `OnGatewayInit, OnGatewayConnection, OnGatewayDisconnect`
    - Constructor injects `JwtService` (from AuthModule) and `PrismaService`
    - `afterInit(server: Server)`: registers `server.use(async (socket, next) => {...})` middleware that reads `socket.handshake.auth?.token`, calls `this.jwtService.verify(token)`, attaches payload to `socket.data.jwtPayload`, calls `next(new Error('...'))` on failure. NEVER throws — always calls next().
    - `handleConnection(client: Socket)`: reads `client.data.jwtPayload`; if absent calls `client.disconnect(true)` and returns. Otherwise resolves vendorId via `prisma.vendorProfile.findUnique({ where: { userId: payload.sub } })` — use `payload.sub` for userId (matches JWT strategy). If vendor not found, disconnect. Join room `vendor:${vendor.id}`, set `client.data.vendorId`.
    - `handleDisconnect(client: Socket)`: logs vendorId.
    - `emitToVendor(vendorId: string, event: string, data: unknown): void` — calls `this.server.to(\`vendor:${vendorId}\`).emit(event, data)`. This is the method RoutingService will call in Plan 04-03.

    Note: Check JWT payload shape from Phase 1 JwtStrategy — the sub field is userId. Verify with:
    ```bash
    grep -r "sub\|userId\|payload" api/src/auth/jwt.strategy.ts
    ```

    **File: api/src/inbox/inbox.service.ts**
    Injectable service with PrismaService and RedisService injected.

    Method `acceptLead(assignmentId: string, vendorId: string)`:
    - Uses `prisma.$transaction(async (tx) => {...})`
    - `tx.leadAssignment.updateMany({ where: { id: assignmentId, vendorId, status: 'NOTIFIED' }, data: { status: 'ACCEPTED', respondedAt: new Date() } })` — if count === 0 throw BadRequestException
    - Fetch assignment with `tx.leadAssignment.findUnique({ where: { id: assignmentId }, include: { lead: { include: { customer: { select: { phone: true, id: true } } } } } })`
    - `tx.consentLog.create({ data: { userId: customer.id, consentType: 'PHONE_REVEAL', status: 'GRANTED', metadata: { revealedToVendorId: vendorId } } })`
    - Outside transaction: `redis.del(\`vendor:score:factors:${vendorId}\`)` (invalidate scoring cache — avoids long-running TX per pitfall 4)
    - Return `{ assignmentId, phone: customer.phone, leadId: assignment.leadId }`

    Method `declineLead(assignmentId: string, vendorId: string, reason: string)`:
    - `prisma.leadAssignment.updateMany({ where: { id: assignmentId, vendorId, status: 'NOTIFIED' }, data: { status: 'DECLINED', respondedAt: new Date() } })` — if count === 0 throw BadRequestException
    - `redis.del(\`vendor:score:factors:${vendorId}\`)` (invalidate cache)
    - Return `{ assignmentId, status: 'DECLINED' }`

    Method `getInbox(vendorId: string, page: number, limit: number)`:
    - `prisma.leadAssignment.findMany({ where: { vendorId }, include: { lead: { select: { id: true, eventType: true, eventDate: true, city: true, budget: true, status: true, quotes: { where: { vendorId }, select: { id: true, status: true, totalPaise: true } } } } }, orderBy: { createdAt: 'desc' }, skip: (page-1)*limit, take: limit })`
    - Return paginated list with assignment status and quote status if quote exists
    - IMPORTANT: Do NOT include customer.phone in inbox response — only revealed via acceptLead
