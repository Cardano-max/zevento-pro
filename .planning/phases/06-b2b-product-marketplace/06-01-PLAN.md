**1. Create DTOs** in `api/src/product/dto/`:

- **CreateProductDto**: `name` (IsString, MaxLength 200), `categoryId` (IsUUID), `pricePaise` (IsInt, Min 100), `stock` (IsInt, Min 0), `lowStockThreshold` (IsOptional, IsInt, Min 0, default 5), `moq` (IsOptional, IsInt, Min 1, default 1), `description` (IsOptional, IsString), `fulfillmentSource` (IsOptional, IsEnum FulfillmentSource). All validation via class-validator decorators.

- **UpdateProductDto**: All fields optional (PartialType of CreateProductDto using `@nestjs/mapped-types`). Also add `isActive` (IsOptional, IsBoolean).

- **SearchProductsDto**: `search` (IsOptional, IsString), `categoryId` (IsOptional, IsUUID), `priceMin` (IsOptional, @Type Number, IsInt, Min 0), `priceMax` (IsOptional, @Type Number, IsInt, Min 0), `vendorId` (IsOptional, IsUUID), `page` (IsOptional, @Type Number, IsInt, Min 1, default 1), `limit` (IsOptional, @Type Number, IsInt, Min 1, Max 50, default 20). Follow SearchVendorsDto pattern exactly for page/limit.

**2. Create ProductService** (`api/src/product/product.service.ts`):

Inject PrismaService, CloudinaryService, and a BullMQ Queue ('stock-alerts').

Methods:
- `createProduct(vendorId: string, dto: CreateProductDto)`: Create product with `prisma.product.create`. Return the product with category included.

- `updateProduct(vendorId: string, productId: string, dto: UpdateProductDto)`: Find product by id, verify `product.vendorId === vendorId` (Pitfall 4 from research), then update. Throw NotFoundException if not found, ForbiddenException if not owner.

- `deleteProduct(vendorId: string, productId: string)`: Verify ownership. Delete all ProductImage records and their Cloudinary assets (loop + `cloudinaryService.deleteImage`) BEFORE deleting the product itself. Use `$transaction` for atomicity: fetch images, delete Cloudinary assets (outside tx), then inside tx delete images and product together. This prevents Cloudinary orphans (Pitfall 5).

- `addImage(vendorId: string, productId: string, file: Express.Multer.File)`: Verify ownership. Upload via `cloudinaryService.uploadImage(file, 'product-images')`. Create ProductImage record. Return the image record.

- `deleteImage(vendorId: string, imageId: string)`: Find image with product included, verify product ownership, delete from Cloudinary, delete the ProductImage record.

- `adjustStock(vendorId: string, productId: string, adjustment: number)`: Verify ownership. Update stock using `prisma.product.update` with `stock: { increment: adjustment }` (adjustment can be negative for manual decrements, positive for restocking). After update, check if new stock <= lowStockThreshold and if so, enqueue a BullMQ job to 'stock-alerts' queue with `{ productId, currentStock: newStock }`.

- `getMyProducts(vendorId: string, page: number, limit: number)`: Paginated list of vendor's own products with category and first image.

- `getProductById(productId: string)`: Full product details with all images, category, vendor info.

**3. Create CatalogService** (`api/src/product/catalog.service.ts`):

Inject PrismaService.

Methods:
- `searchProducts(dto: SearchProductsDto)`: Build `where` filter: `isActive: true`, vendor must have `status: 'APPROVED'` and subscription `status: { in: ['ACTIVE', 'AUTHENTICATED'] }`. If `dto.search`, filter by `name: { contains: dto.search, mode: 'insensitive' }`. If `dto.categoryId`, filter by categoryId. If `dto.priceMin`, filter `pricePaise: { gte: dto.priceMin }`. If `dto.priceMax`, filter `pricePaise: { lte: dto.priceMax }`. If `dto.vendorId`, filter by vendorId. Use findMany with skip/take pagination, select product fields + first image (thumbnail) + category name + vendor businessName. Return `{ data, pagination: { page, limit, total, totalPages } }`. Follow CustomerService.searchVendors pattern exactly.

- `getProductDetail(productId: string)`: Find product by id with `isActive: true`, include ALL images (ordered by sortOrder), category, vendor (businessName, id). Throw NotFoundException if not found or inactive.

- `getCategories()`: Return all active ProductCategories ordered by sortOrder, with children included (one level deep).

**4. Create ProductController** (`api/src/product/product.controller.ts`):

`@Controller('products')` with `@UseGuards(JwtAuthGuard, RolesGuard, VendorOwnerGuard)` and `@Roles('SUPPLIER')` at class level. This ensures only SUPPLIERs can manage products (Pitfall 6).

Endpoints:
- `POST /products` -> `createProduct(req.vendorId, dto)` — Create a new product
- `PATCH /products/:id` -> `updateProduct(req.vendorId, id, dto)` — Update product
- `DELETE /products/:id` -> `deleteProduct(req.vendorId, id)` — Delete product + images
- `GET /products/mine` -> `getMyProducts(req.vendorId, page, limit)` — List supplier's own products (paginated)
- `POST /products/:id/images` -> `addImage(req.vendorId, id, file)` — Upload product image (FileInterceptor with memoryStorage, 10MB limit, imageFileFilter from vendor.controller pattern)
- `DELETE /products/images/:imageId` -> `deleteImage(req.vendorId, imageId)` — Delete product image
- `PATCH /products/:id/stock` -> `adjustStock(req.vendorId, id, body.adjustment)` — Manual stock adjustment (for restocking or corrections). Body: `{ adjustment: number }` (positive = add, negative = subtract).

Use ParseUUIDPipe for all id params. Use @Req() for req.vendorId (set by VendorOwnerGuard).

**Define imageFileFilter inline** (same pattern as vendor.controller.ts lines 34-48): accept JPEG, PNG, WebP only.

**5. Create CatalogController** (`api/src/product/catalog.controller.ts`):

- `GET /catalog/products/:id` -> `getProductDetail(id)` — Product detail (public)

**6. Create StockAlertProcessor** (`api/src/product/processor/stock-alert.processor.ts`):

`@Processor('stock-alerts')` BullMQ worker. Inject PrismaService and NotificationService. On job, fetch product with vendor info. If `currentStock <= product.lowStockThreshold`, call `notificationService.sendPushToVendor(product.vendorId, { leadId: product.id, eventType: \`Low stock: ${product.name} (${job.data.currentStock} remaining)\`, city: 'Inventory Alert' })`. Follow PaymentProcessor pattern for error handling and `@OnWorkerEvent('failed')`.
