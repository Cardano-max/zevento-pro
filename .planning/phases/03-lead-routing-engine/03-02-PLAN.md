1. Create `api/src/lead/types/vendor-score.interface.ts`:
   ```typescript
   export interface VendorScoreFactors {
     vendorId: string;
     subscriptionTier: string;  // 'BASIC' | 'PREMIUM'
     averageRating: number;     // 0-5
     responseRate: number;      // 0-1
     locationMatch: boolean;
     fairnessCount: number;     // leads in current rotation window
   }
   ```

2. Create `api/src/lead/scoring.service.ts`:
   - Inject PrismaService, RedisService
   - Constants: WEIGHTS object (SUBSCRIPTION_TIER: 0.30, RATING: 0.20, RESPONSE_RATE: 0.20, LOCATION_MATCH: 0.20, FAIRNESS_ROTATION: 0.10), SCORE_CACHE_TTL = 300, FAIRNESS_WINDOW_DAYS = 7, MAX_LEADS_PER_WINDOW = 50
   - `computeScore(factors: VendorScoreFactors): number` — pure function implementing the weighted formula:
     - tierScore: PREMIUM=1.0, BASIC=0.5
     - ratingScore: averageRating / 5.0
     - responseScore: responseRate (already 0-1)
     - locationScore: locationMatch ? 1.0 : 0.0
     - fairnessScore: Math.max(0, 1.0 - (fairnessCount / MAX_LEADS_PER_WINDOW))
     - Return weighted sum
   - `findVendorsInRange(latitude: number, longitude: number, categoryId?: string): Promise<string[]>` — use Prisma $queryRaw with PostGIS:
     - Query vendor_service_areas JOIN markets WHERE ST_DWithin(ST_MakePoint(m.longitude, m.latitude)::geography, ST_MakePoint($longitude, $latitude)::geography, vsa.radius_km * 1000)
     - Also join vendor_profiles WHERE status='APPROVED'
     - Also join vendor_subscriptions WHERE status IN ('ACTIVE','AUTHENTICATED')
     - If categoryId provided, join vendor_categories WHERE category_id matches
     - Return array of vendor_id strings
     - Use snake_case column names in raw SQL (per Pitfall 3 from research)
   - `getScoreFactors(vendorId: string, eventLat: number, eventLng: number): Promise<VendorScoreFactors>`:
     - Check Redis cache: key `vendor:score:factors:${vendorId}`
     - If cached, parse JSON, override locationMatch with fresh PostGIS check for this specific event location
     - If not cached, query: VendorProfile with subscription (plan tier), vendorStats (averageRating, responseRate), then cache the non-location factors with TTL
     - Get fairness count from Redis: key `fairness:${vendorId}` (INCR'd on assignment, 7-day TTL)
     - Return VendorScoreFactors
   - `scoreVendors(vendorIds: string[], eventLat: number, eventLng: number): Promise<Array<{vendorId: string, score: number}>>`:
     - For each vendorId, get score factors and compute score
     - Sort descending by score
     - Return sorted array
   - `invalidateScoreCache(vendorId: string): Promise<void>` — delete Redis key `vendor:score:factors:${vendorId}`

   IMPORTANT: Use `::geography` cast on ST_MakePoint for meter-based distances (Pitfall 7 from research).
   IMPORTANT: Use snake_case column names in $queryRaw (Pitfall 3).
  </action>

  </verify>
