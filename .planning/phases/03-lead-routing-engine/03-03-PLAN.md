1. Create `api/src/routing/routing.constants.ts`:
   - LEAD_ROUTING_QUEUE = 'lead-routing'
   - ROUTE_LEAD_JOB = 'route-lead'
   - TOP_N = 3 (number of vendors for Mode B)
   - FAIRNESS_WINDOW_SECONDS = 7 * 24 * 60 * 60 (7 days)

2. Create `api/src/routing/routing.service.ts`:
   - Inject PrismaService, ScoringService, NotificationService, RedisService
   - `routeDirect(leadId: string)` — Mode A:
     a. Load lead with targetVendorId
     b. Create LeadAssignment record: { leadId, vendorId: lead.targetVendorId, score: null, status: 'PENDING' }
     c. Update lead status to 'ROUTED'
     d. Increment fairness counter: Redis INCR `fairness:${vendorId}` with FAIRNESS_WINDOW_SECONDS TTL (use `incr` then `expire` only if TTL is -1 to avoid resetting window)
     e. Update LeadAssignment notifiedAt, status to 'NOTIFIED'
     f. Send push notification via NotificationService
   - `routeTopThree(leadId: string)` — Mode B:
     a. Load lead with lat/lng and categoryId
     b. Update lead status to 'ROUTING'
     c. Call ScoringService.findVendorsInRange(lead.latitude, lead.longitude, lead.categoryId) to get eligible vendor IDs
     d. If no vendors found, update lead status to 'ROUTED' (with 0 assignments) and return
     e. Call ScoringService.scoreVendors(vendorIds, lead.latitude, lead.longitude)
     f. Apply fairness cap: for each vendor in scored order, check Redis `fairness:${vendorId}` — if count >= MAX_LEADS_PER_WINDOW (50), skip. Select top N (3) vendors that pass the cap.
     g. Create LeadAssignment records for selected vendors with their scores
     h. Update lead status to 'ROUTED'
     i. For each assigned vendor: increment fairness counter (INCR + conditional EXPIRE), update notifiedAt, set status to 'NOTIFIED'
     j. Send push notifications to all assigned vendors via NotificationService.sendPushToMultipleVendors

3. Create `api/src/routing/routing.processor.ts`:
   - Extend WorkerHost from @nestjs/bullmq
   - Decorate with @Processor(LEAD_ROUTING_QUEUE)
   - Inject RoutingService
   - `process(job: Job<{ leadId: string; mode: 'A' | 'B' }>)`:
     - If mode === 'A': await routingService.routeDirect(job.data.leadId)
     - If mode === 'B': await routingService.routeTopThree(job.data.leadId)
   - Add @OnWorkerEvent('failed') handler: log error with job.id and error message

4. Create `api/src/routing/routing.module.ts`:
