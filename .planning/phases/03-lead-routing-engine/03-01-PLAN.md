  <action>
1. Add to `packages/shared/src/enums.ts`:
   - `LeadStatus` enum: PENDING, ROUTING, ROUTED, EXPIRED, CANCELLED
   - `LeadAssignmentStatus` enum: PENDING, NOTIFIED, ACCEPTED, DECLINED, EXPIRED

2. Add to `api/prisma/schema.prisma` (Phase 3 section after Phase 2 models):

   **VendorStats model** — cached scoring factors per vendor:
   - id (UUID PK), vendorId (unique, FK to VendorProfile), averageRating (Float, default 3.0), responseRate (Float, default 0.5), totalLeadsReceived (Int, default 0), totalLeadsWon (Int, default 0), updatedAt
   - Map to `vendor_stats`, index on vendorId

   **Lead model** — customer event inquiry:
   - id (UUID PK), customerId (FK to User), eventType (VarChar 50), eventDate (DateTime), city (VarChar 100), latitude (Float?), longitude (Float?), budget (Int, in paise), guestCount (Int), targetVendorId (UUID?, FK to VendorProfile for Mode A), categoryId (UUID?, FK to EventCategory for Mode B), status (VarChar 20, default PENDING), consentLogId (UUID), createdAt, updatedAt
   - Relations: customer (User), targetVendor (VendorProfile?), category (EventCategory?), assignments (LeadAssignment[])
   - Indexes: customerId, status, categoryId
   - Map to `leads`

   **LeadAssignment model** — vendor assigned to lead:
   - id (UUID PK), leadId (FK to Lead), vendorId (FK to VendorProfile), score (Float?), status (VarChar 20, default PENDING), notifiedAt (DateTime?), respondedAt (DateTime?), createdAt
   - Unique constraint: [leadId, vendorId]
   - Indexes: vendorId, status
   - Map to `lead_assignments`

   **DeviceToken model** — FCM tokens for push notifications:
   - id (UUID PK), userId (FK to User), token (unique, VarChar 255), platform (VarChar 20), isActive (Boolean, default true), createdAt, updatedAt
   - Index on userId
   - Map to `device_tokens`

3. Add reverse relations on existing models:
   - User: `leads Lead[]` (relation "CustomerLeads"), `deviceTokens DeviceToken[]`
   - VendorProfile: `stats VendorStats?`, `leadAssignments LeadAssignment[]`, `targetedLeads Lead[]`
   - EventCategory: `leads Lead[]`

4. Create a raw SQL migration for PostGIS extension. Run `npx prisma migrate dev --name phase3-lead-routing` which will generate the migration, then PREPEND to the generated migration.sql file:
   ```sql
   CREATE EXTENSION IF NOT EXISTS postgis;
   ```
   This ensures PostGIS functions (ST_DWithin, ST_MakePoint) are available for distance queries.

5. Run `npx prisma generate` to update the client.

6. Update `api/prisma/seed.ts` to create a default VendorStats record for any existing vendor profiles (with default averageRating=3.0, responseRate=0.5).

   - Use class-validator: @IsOptional, @IsUUID, @IsString, @IsInt, @Min, @Max, @Type (from class-transformer for query param number coercion)

2. Create `api/src/customer/customer.service.ts`:
   - Inject PrismaService
   - `listCategories()`: Find all active EventCategories ordered by sortOrder. Include children for hierarchy. Return id, name, slug, description, sortOrder.
   - `searchVendors(dto: SearchVendorsDto)`: Query VendorProfile where status='APPROVED' and vendor has active subscription (VendorSubscription with status in ['ACTIVE','AUTHENTICATED']). Filter by:
     - categoryId: join VendorCategory where categoryId matches
     - city: join VendorServiceArea -> Market where market.city = dto.city (case-insensitive)
     - budgetMin/budgetMax: pricingMin >= budgetMin AND/OR pricingMax <= budgetMax
     - Paginate with skip/take. Return vendor profiles with businessName, description, pricingMin, pricingMax, categories (names), serviceAreas (city names), stats (averageRating), portfolio photo count. Include total count for pagination.
   - `getVendorProfile(vendorId: string)`: Find VendorProfile by id where status='APPROVED'. Include categories (with EventCategory name/slug), photos (ordered by sortOrder), serviceAreas (with Market city/state), stats (averageRating, responseRate), subscription plan tier. Throw NotFoundException if not found or not approved.

3. Create `api/src/customer/customer.controller.ts`:
   - NO auth guards — all endpoints are public
   - `GET /customer/categories` → calls listCategories()
   - `GET /customer/vendors` → calls searchVendors(dto) with @Query() validation
   - `GET /customer/vendors/:id` → calls getVendorProfile(id) with @Param('id', ParseUUIDPipe)

4. Create `api/src/customer/customer.module.ts`:

   - Provide CustomerService, register CustomerController
