**PaymentWebhookController** (`webhook/payment-webhook.controller.ts`):
- `@Controller('webhooks/razorpay')` -- separate from the existing subscription webhook controller at `webhooks/razorpay/subscription`
- NO auth guards (public endpoint -- Razorpay calls this)
- `POST /webhooks/razorpay/payment`:
  - Read raw body from `@Req() req` using `req.rawBody` (rawBody is already enabled in main.ts from Phase 2)
  - Read signature from `@Headers('x-razorpay-signature') signature: string`
  - Call `paymentWebhookService.handleWebhook(rawBody, signature)`
  - Wrap in try/catch: catch logs error, always returns `{ status: 'ok' }` (existing pattern: return 200 to prevent Razorpay retry storms)

**PaymentWebhookService** (`webhook/payment-webhook.service.ts`):
- Inject `PrismaService`, `RazorpayService`, BullMQ queue (`@InjectQueue('payment-processing')`)
- `handleWebhook(rawBody: string, signature: string)`:
  1. Verify signature using `razorpayService.validateWebhookSignature(rawBody, signature, process.env.RAZORPAY_WEBHOOK_SECRET)`. If invalid, throw UnauthorizedException.
  2. Parse payload: `JSON.parse(rawBody)`. Extract `event`, `payload.payment.entity` as paymentEntity, `payload.order.entity` as orderEntity (or null).
  3. Build idempotency key: `externalId = '${paymentEntity.id}_${event}'` (e.g., `pay_ABC123_payment.captured`).
  4. Create webhookEvent in DB (P2002 catch for duplicate -- identical pattern to SubscriptionWebhookService):
     ```typescript
     try {
       webhookEvent = await this.prisma.webhookEvent.create({
         data: { provider: 'RAZORPAY', externalId, eventType: event, status: 'RECEIVED', payload }
       });
     } catch (error: any) {
       if (error?.code === 'P2002') { this.logger.log(`Duplicate payment event skipped: ${externalId}`); return; }
       throw error;
     }
     ```
  5. Route by event type:
     - `payment.captured`: Enqueue BullMQ job `payment-processing` with `{ webhookEventId: webhookEvent.id, paymentEntity, orderNotes: paymentEntity.notes || {} }`. Use 3 attempts with exponential backoff (60s base).
     - `payment.failed`: Update booking paymentStatus to FAILED directly (find booking by razorpayOrderId from paymentEntity.order_id). Mark webhookEvent PROCESSED.
     - `refund.processed`: Update the matching Transaction record's status to REFUNDED and the booking's paymentStatus to REFUNDED. Mark webhookEvent PROCESSED.
     - Default: log warning, mark webhookEvent PROCESSED.

**PaymentProcessor** (`processor/payment.processor.ts`):
- `@Processor('payment-processing')` extends WorkerHost
- Inject `PrismaService`, `CommissionService` (from Plan 05-01)
- `process(job)`:
  1. Extract `webhookEventId`, `paymentEntity`, `orderNotes` from `job.data`
  2. Find booking by `razorpayOrderId: paymentEntity.order_id`
  3. If no booking found, log error and mark webhookEvent FAILED, return
  4. Get commission rate: prefer `booking.commissionRateBps` (locked at order time). If null (edge case), fall back to `commissionService.getRate(booking.vendorId, categoryId)`
  5. Calculate split:
     ```typescript
     const totalPaise = paymentEntity.amount;
     const commissionPaise = Math.round((totalPaise * commissionRateBps) / 10000);
     const netPayoutPaise = totalPaise - commissionPaise;
     ```
  6. Create Transaction record in prisma:
     ```typescript
     await this.prisma.transaction.create({
       data: {
         bookingId: booking.id,
         type: 'BOOKING_COMMISSION',
         amountPaise: totalPaise,
         commissionPaise,
         netPayoutPaise,
         razorpayPaymentId: paymentEntity.id,
         razorpayOrderId: paymentEntity.order_id,
         status: 'PAID',
         paidAt: new Date(),
       },
     });
     ```
  7. Update booking: `paymentStatus = 'CAPTURED'`
  8. Mark webhookEvent as PROCESSED

**PayoutService** (`payout.service.ts`):
- Inject `PrismaService`
- Constructor: Read `RAZORPAY_X_KEY_ID`, `RAZORPAY_X_KEY_SECRET`, `RAZORPAY_X_ACCOUNT_NUMBER` from env. If any missing, set `devMode = true` and log warning (consistent with RazorpayService dev mock pattern).
- `createPayout(params: { bookingId: string; vendorId: string; netPayoutPaise: number; razorpayPaymentId: string })`:
  1. Fetch vendor: `prisma.vendorProfile.findUnique({ where: { id: params.vendorId }, include: { user: { select: { phone: true, email: true } } } })`
  2. Check bank details: if `!vendor.bankAccountNumber || !vendor.bankIfsc || !vendor.bankAccountName`, update Transaction `payoutStatus = 'PENDING_BANK_DETAILS'` and return `{ status: 'PENDING_BANK_DETAILS' }`. Do NOT throw -- the payout can be retried later when bank details are added.
  3. If dev mock mode:
     ```typescript
     this.logger.warn(`Dev mode: mock payout of ${params.netPayoutPaise} paise to vendor ${params.vendorId}`);
     return { id: `pout_mock_${Date.now()}`, status: 'processing', amount: params.netPayoutPaise };
     ```
  4. Build and send RazorpayX Composite Payout API request:
     ```typescript
     const response = await fetch('https://api.razorpay.com/v1/payouts', {
       method: 'POST',
       headers: {
         'Content-Type': 'application/json',
         'Authorization': `Basic ${Buffer.from(`${this.xKeyId}:${this.xKeySecret}`).toString('base64')}`,
         'X-Payout-Idempotency': `payout_${params.bookingId}_${params.razorpayPaymentId}`,
       },
       body: JSON.stringify({
         account_number: this.accountNumber,
         fund_account: {
           account_type: 'bank_account',
           bank_account: {
             name: vendor.bankAccountName,
             ifsc: vendor.bankIfsc,
             account_number: vendor.bankAccountNumber,
           },
           contact: {
             name: vendor.businessName,
             type: 'vendor',
             email: vendor.user.email || undefined,
             contact: vendor.user.phone,
           },
         },
         amount: params.netPayoutPaise,
         currency: 'INR',
         mode: 'IMPS',
         purpose: 'vendor_payout',
         queue_if_low_balance: true,
         reference_id: `booking_${params.bookingId}`.substring(0, 40),
         narration: `Zevento payout for booking ${params.bookingId.substring(0, 8)}`,
       }),
     });
     ```
  5. Parse response. If HTTP error, throw with details. Return payout response object.

**PayoutProcessor** (`processor/payout.processor.ts`):
- `@Processor('vendor-payout')` extends WorkerHost
- Inject `PrismaService`, `PayoutService`
- `process(job)`:
  1. Extract `bookingId`, `vendorId`, `netPayoutPaise`, `razorpayPaymentId` from `job.data`
  2. Verify booking is COMPLETED: `prisma.booking.findUnique({ where: { id: bookingId } })`. If status !== 'COMPLETED', log info and throw to retry later (BullMQ will retry with backoff). This handles the race condition where the payout job fires before the booking transitions to COMPLETED.
  3. Call `payoutService.createPayout({ bookingId, vendorId, netPayoutPaise, razorpayPaymentId })`
  4. Update Transaction: find by bookingId and type BOOKING_COMMISSION, update `razorpayPayoutId = result.id`, `payoutStatus = result.status === 'processing' ? 'PROCESSING' : 'QUEUED'`
  5. Log success

**Wire payout trigger into BookingService** (`booking.service.ts`):
- Inject `@InjectQueue('vendor-payout') private readonly payoutQueue: Queue` -- add BullMQ import
- In `transitionStatus` method, after the `$transaction` that transitions booking to COMPLETED:
  ```typescript
  if (toStatus === 'COMPLETED') {
    // Find the payment transaction for this booking
    const tx = await this.prisma.transaction.findFirst({
      where: { bookingId: booking.id, type: 'BOOKING_COMMISSION', status: 'PAID' },
    });
    if (tx && tx.netPayoutPaise) {
      await this.payoutQueue.add('vendor-payout', {
        bookingId: booking.id,
        vendorId: booking.vendorId,
        netPayoutPaise: tx.netPayoutPaise,
        razorpayPaymentId: tx.razorpayPaymentId,
      }, {
        attempts: 5,
        backoff: { type: 'exponential', delay: 60000 },
      });
    }
  }
  ```
  This ensures payout is only triggered after booking COMPLETED, not on payment capture (research pitfall 5).
