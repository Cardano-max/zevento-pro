**CommissionService** (`commission.service.ts`):
- Inject `PrismaService`
- `getRate(vendorId: string, categoryId?: string): Promise<number>`:
  1. Fetch `vendorProfile.role` (PLANNER or SUPPLIER) from prisma
  2. Query `commissionRate.findFirst` with specificity cascade:
     - WHERE `effectiveFrom <= now` AND (`effectiveTo IS NULL OR effectiveTo >= now`)
     - ORDER BY: categoryId DESC NULLS LAST, vendorRole DESC NULLS LAST
     - Filter: `(categoryId AND vendorRole) OR (categoryId AND vendorRole IS NULL) OR (categoryId IS NULL AND vendorRole) OR (categoryId IS NULL AND vendorRole IS NULL)`
     - Use Prisma `findMany` with orderBy to get most specific first, take first result
  3. If no rate found, throw `InternalServerErrorException('No commission rate configured')`
  4. Return `rate.rateBps`

**PaymentService** (`payment.service.ts`):
- Inject `PrismaService`, `RazorpayService` (from SubscriptionModule), `CommissionService`
- `createBookingOrder(bookingId: string, userId: string)`:
  1. Find booking with `include: { quote: true, vendor: { select: { id: true, businessName: true } } }`
  2. Validate: booking exists, booking.customerId === userId, booking.status === 'BOOKED', booking.paymentStatus is null or 'FAILED' (allow retry)
  3. If booking already has a razorpayOrderId and paymentStatus is not FAILED, throw BadRequest('Payment already initiated')
  4. Get commission rate: `commissionService.getRate(booking.vendorId, booking.lead?.categoryId)` -- need to include lead relation for categoryId. Alternative: store categoryId on booking. Simpler: include lead in query.
     Actually, looking at Booking model -- it has leadId. Include lead to get categoryId:
     ```
     include: { quote: true, vendor: { select: { id: true, businessName: true } }, lead: { select: { categoryId: true } } }
     ```
  5. Create Razorpay order via `razorpayService` -- add `createOrder` method to RazorpayService OR call SDK directly. Since RazorpayService is in SubscriptionModule and we need to add `createOrder` and `createRefund` methods, extend RazorpayService with:
     - `createOrder({ amount, currency, receipt, notes })` -- calls `this.instance!.orders.create(...)`, dev mock returns `{ id: 'order_mock_...', amount, currency, receipt, status: 'created' }`
     - `validatePaymentSignature(orderId, paymentId, signature)` -- HMAC SHA256 of `orderId|paymentId` with key_secret, returns boolean
     - `createRefund(paymentId, { amount?, notes? })` -- calls `this.instance!.payments.refund(paymentId, ...)`, dev mock returns mock refund object
  6. Update booking: `razorpayOrderId = order.id`, `paymentStatus = 'PENDING'`, `commissionRateBps = rateBps`
  7. Return `{ orderId: order.id, amount: order.amount, currency: 'INR', keyId: process.env.RAZORPAY_KEY_ID || 'rzp_test_mock' }`

- `verifyPayment(dto: VerifyPaymentDto)`:
  1. Verify signature: HMAC SHA256 of `dto.razorpay_order_id|dto.razorpay_payment_id` with RAZORPAY_KEY_SECRET. Use `razorpayService.validatePaymentSignature(...)`.
  2. If invalid, throw UnauthorizedException('Invalid payment signature')
  3. Find booking by razorpayOrderId
  4. If booking found, update `paymentStatus = 'CAPTURED'` (optimistic for UX -- webhook is source of truth)
  5. Return `{ status: 'ok', paymentId: dto.razorpay_payment_id }`
